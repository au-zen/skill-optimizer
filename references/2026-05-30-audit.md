## Summary

Full SkillOpt methodology audit of the skill-optimizer skill (SKILL.md + scripts/).
**Score: 6.6/10 ‚Üí ~8.5/10 after fixes.**

## Top-3 Issues Fixed

### 1. Missing Agent-Facing Sections (P1)

**Before:** Frontmatter lacked `trigger`/`rules`. Body had no `## Rules`, `## When Loaded`, or `## Quickstart`.

**Fix:** Added:
- Frontmatter `trigger` (activation signal) and `rules` (4 actionable constraints)
- `## Rules` ‚Äî behavioral rules the agent must follow
- `## When Loaded` ‚Äî agent directives for manual vs auto mode
- `## Quickstart` ‚Äî minimal working example with `--hermes-default`

### 2. Validation Gate Auto-Accept (P1)

**Before:** `_validate_and_gate()` auto-accepted edits (score + 0.01 bump) when no sel split existed, defeating the core protection mechanism.

**Fix:** Returns `False` with an `[ERR]` message requiring `--sel` or `--sel-trajs`.

### 3. Missing Structural Validation (P3)

**Before:** `_apply_edits()` wrote candidate files without checking frontmatter, code fence balance, or section headings.

**Fix:** New `_validate_skill_structure()` checks:
- YAML frontmatter exists and is non-empty
- Fenced code blocks are paired (even ` ``` ` count)
- At least one `## `-level section heading present

### Key Resolution (critical fix)

**Problem:** `OPENROUTER_API_KEY` was found first in `.env` (alphabetical order) and sent to DeepSeek API endpoint ‚Üí 401.

**Fix:** `_resolve_api_key(model_hint)` skips non-deepseek keys when model contains "deepseek". Key detection chain:
1. `SKILLOPT_API_KEY` (env var)
2. `DEEPSEEK_API_KEY` (env var) ‚Äî **preferred for deepseek models**
3. `.env` file scanning ‚Äî **skips non-deepseek keys when model is deepseek**
