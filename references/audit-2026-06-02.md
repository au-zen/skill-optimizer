## 🔴 Logic Issues (potential runtime breakage)

### `merge_trajs.py:72` — `id(t)` as synthetic task ID
```python
t["task_id"] = f"anon_{id(t)}"  # id() not stable across GC cycles
```
`id()` is unique only for the object's lifetime. If the dict holding the
trajectory reference is garbage-collected and a new trajectory object gets
the same address, dedup silently collapses them.

**Fix:** UUID (`uuid.uuid4().hex[:8]`) or a monotonic counter.

### `__init__.py:8-12` — `score_skill` silently becomes None on ImportError
```python
try:
    from .score_skill import score_skill, check_stability
except ImportError:
    score_skill = None  # still exported in __all__
```
When `yaml` is missing, `score_skill` is set to None but still exported.
Callers get None and hit AttributeError on `.get()`.

**Fix:** Don't export None symbols; add a proxy that raises on use.
